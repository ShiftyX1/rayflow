# Системы освещения Rayflow - Краткий справочник

> **Полная документация на английском:** [LIGHTING_SYSTEMS.md](LIGHTING_SYSTEMS.md)

## TL;DR

В Rayflow используются **две разные системы освещения**:

| Система | Что это | Назначение |
|---------|---------|------------|
| **`BlockType::Light`** | Блок-источник света | Статическое освещение мира (Minecraft-стиль) |
| **`PointLight`** | Динамический 3D источник | Освещение игроков и движущихся объектов |

**Они НЕ заменяют друг друга!** Обе системы работают параллельно.

---

## 1. BlockType::Light - Блочное освещение

### Что это?
Тип блока (как камень, дерево, и т.д.), который **излучает свет**.

### Где используется?
- **Map Editor** - размещение постоянных источников света на картах
- **Статическое освещение** - факелы, лампы, лава
- **Детерминированное освещение** - одинаковое на всех клиентах

### Как работает?
```cpp
// В shared/block.hpp
enum class BlockType {
    Light,  // Блок-источник света (emission = 15)
};

BlockLightProps{ 15u, 0u, 0u, false, false }
//               ^^^ излучает свет уровня 15
```

Алгоритм:
1. Свет распространяется от блока по соседям (BFS)
2. Уменьшается на 1 каждый шаг: 15 → 14 → 13 → ... → 0
3. Результат сохраняется в вершинах меша (vertex colors)
4. Нагрузка на сервер: **НОЛЬ** (расчёт на клиенте)

---

## 2. PointLight - Динамическое освещение

### Что это?
Структура данных для **динамических 3D источников света** в шейдерах.

### Где используется?
- **Освещение игроков** - "ореол" света вокруг персонажа
- **Временные эффекты** - взрывы, магия, выстрелы
- **Динамические объекты** - движущиеся светильники

### Как работает?
```cpp
// В client/world.hpp
struct PointLight {
    Vector3 position;   // 3D позиция
    Vector3 color;      // RGB цвет
    float radius;       // Радиус
    float intensity;    // Интенсивность
};
```

Алгоритм:
1. Каждый кадр передаётся массив PointLight в shader
2. В fragment shader для каждого пикселя:
   - Вычисляется расстояние до источников
   - Добавляется вклад света с затуханием
3. Лимит: **32 источника** (MAX_LIGHTS)

---

## Сравнение

| Характеристика | BlockType::Light | PointLight |
|----------------|------------------|------------|
| **Тип** | Блок в мире | Struct в памяти |
| **Расчёт** | BFS на CPU | Per-pixel на GPU |
| **Обновление** | При изменении блоков | Каждый кадр |
| **Лимит** | Неограниченно | 32 штуки |
| **Детерминизм** | Да | Нет |
| **Использование** | Статика | Динамика |

---

## Как они работают вместе?

### В шейдере (voxel.fs):

```glsl
// 1. Базовый свет от BlockType::Light (из vertex color)
float skylightFactor = max(fragSkyLight, minAmbient);
vec3 lighting = ambientColor * skylightFactor;

// 2. Добавление динамических PointLight
for (int i = 0; i < lightCount; i++) {
    PointLight light = lights[i];
    // ... расчёт и добавление
    lighting += light.color * light.intensity * att;
}

// 3. Итоговый цвет
finalColor = textureColor * lighting * AO;
```

### Визуально:
- **BlockType::Light** → Общее освещение мира (базовый свет)
- **PointLight** → "Ореол" вокруг движущихся объектов

## Примеры использования

### Пример 1: Игрок в тёмной пещере

```cpp
// Статический факел на стене (BlockType::Light)
world.set_block(10, 64, 10, BlockType::Light);
// → Освещает область вокруг позиции (10, 64, 10)

// Динамический свет от игрока (PointLight)
PointLight player_light{
    .position = player_pos,
    .color = {1.0f, 0.9f, 0.7f},
    .radius = 12.0f,
    .intensity = 1.0f
};
world.set_lights({player_light});
// → Движется вместе с игроком
```

**Результат:**
- Факелы на стенах дают **статичное** базовое освещение
- Свет от игрока добавляет **динамическую** подсветку

### Пример 2: Редактор карт

```cpp
// В редакторе: размещение источника света
// Используется BlockType::Light для детерминированного результата
map_editor.place_block(x, y, z, BlockType::Light);

// В игре: добавление динамического света у игроков
// Используется PointLight для плавного движения
game_client.update_lights(player_positions);
```

## FAQ

### В: Почему две системы, а не одна?

**О:** Производительность + качество:
- **BlockType::Light** очень эффективна для статики (расчёт редко)
- **PointLight** даёт высокое качество для динамики
- Вместе они лучше любой системы по отдельности

### В: Можно ли конвертировать BlockType::Light в PointLight?

**О:** Да, можно автоматически создавать PointLight для каждого Light блока.
Это даст плавное real-time освещение, но будет ограничено 32 источниками.

### В: Какую систему использовать для нового источника света?

**Спросите себя:**
- Источник **статичен** (лампа на стене)? → `BlockType::Light`
- Источник **движется** (игрок, снаряд)? → `PointLight`
- Нужно более **32 источников**? → `BlockType::Light`
- Нужен **детерминизм** (одинаково у всех)? → `BlockType::Light`
- Нужно **высокое качество** для динамики? → `PointLight`
